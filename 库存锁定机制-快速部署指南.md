## ⚠️ 常见问题排查

### 问题 1: 数据库脚本执行失败
**错误**: "Table 'batch_schedule' doesn't exist"
**解决**:
1. 确认 batch_schedule 表已存在: `SHOW TABLES LIKE 'batch_schedule';`
2. 如果不存在，先执行其他数据库初始化脚本
3. 检查外键约束是否正确

### 问题 2: 编译失败 - 找不到类
**错误**: "error: cannot find symbol"
**解决**:
1. 运行 `mvn clean install` 重新下载依赖
2. 检查 pom.xml 中的 mybatis-plus 版本
3. 清除 IDE 缓存: 删除 target 文件夹

### 问题 3: 后端启动失败
**错误**: "Port 8080 is already in use"
**解决**:
```bash
# 方式1: 更改端口
java -jar MES.jar --server.port=8081

# 方式2: 停止占用 8080 的进程
netstat -ano | findstr :8080
taskkill /PID <PID> /F
```

### 问题 4: 前端页面 404
**错误**: 访问 `/material-lock` 显示 404
**解决**:
1. 检查路由是否已配置: `grep -r "material-lock" src/router/`
2. 确认 materialLock.vue 文件存在
3. 重启前端服务: `npm run serve`
4. 清除浏览器缓存: Ctrl + Shift + Delete

### 问题 5: API 返回 401 Unauthorized
**错误**: 调用 API 返回 401
**解决**:
1. 检查是否已登录
2. 在请求头中添加认证 token
3. 查看后端日志中的认证错误
