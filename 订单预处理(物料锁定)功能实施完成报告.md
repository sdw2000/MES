## 一、数据库设计完成

### 1.1 创建的表

#### order_preprocessing - 订单预处理表
```sql
主要字段:
- id: 主键
- order_id: 订单ID
- order_no: 订单号
- order_item_id: 订单明细ID
- material_code: 物料代码
- required_qty: 需求数量(可编辑)
- lock_status: 锁定状态 (未锁定/部分锁定/全部锁定)
- locked_qty: 已锁定数量
- schedule_type: 排程类型 (涂布/复卷/分切)
- status: 预处理状态 (preprocessing/locked/dispatched)
- 时间戳和操作人审计字段
```

#### order_material_lock - 订单物料锁定表
```sql
主要字段:
- id: 主键
- preprocessing_id: 关联预处理记录
- tape_stock_id: 胶带库存ID
- lock_qty: 锁定卷数
- lock_area: 锁定面积(m²)
- fifo_order: FIFO排序号
- lock_status: 锁定状态 (locked/used/released/cancelled)
- version: 乐观锁版本号
```

### 1.2 关键设计特性
✅ 乐观锁机制 - 防止库存并发修改
✅ FIFO排序 - 按先进先出顺序选择库存
✅ 一物多单支持 - 一卷物料可被多个订单锁定

### 2.2 实现的核心功能

#### 物料查询 (FIFO)
- 按物料代码查询所有可用库存
- 按sequence_no排序实现FIFO
- 显示每卷的总数/已锁定数/可锁定数

#### 物料锁定
- 支持一个订单锁定多卷物料
- 自动更新tape_stock的reserved_area
- 乐观锁防并发修改

#### 排程类型自动判断
```
- 无库存 → "涂布"排程
- 母卷 → "复卷"排程
- 复卷/分切卷 → "分切"排程
```

#### 路由分离
```
- 已选库存 → production_schedule (生产排程)
- 无库存 → pending_coating_order_pool (涂布待排池)
```

## 六、关键实现细节

### 6.1 FIFO排序实现
```sql
-- 查询时按 sequence_no 排序
SELECT * FROM tape_stock 
WHERE material_code = ? AND status = 1
ORDER BY sequence_no ASC, id ASC
```

### 6.2 一物多单实现
```sql
-- order_material_lock 可多条记录指向同一 tape_stock
-- tape_stock.reserved_area 记录总锁定面积
-- 锁定时检查: available_area >= lock_area
```

### 6.3 乐观锁实现
```sql
-- 更新时验证版本号
UPDATE tape_stock 
SET reserved_area = ?, version = version + 1
WHERE id = ? AND version = ?
-- 返回受影响行数判断是否成功
```

### 6.4 排程类型自动判断
```java
// 查询预处理的物料
List<OrderMaterialLock> locks = selectByPreprocessingId(preprocessingId);

if (locks.isEmpty()) {
    return "涂布";  // 无库存
}

TapeStock tape = selectById(locks.get(0).getTapeStockId());
if ("母卷".equals(tape.getRollType())) {
    return "复卷";
} else {
    return "分切";
}
```

## 七、已知限制和未来优化

### 7.1 当前限制
- 排程类型只能根据第一条锁定物料判断 (假设同订单物料类型一致)
- 不支持解锁已锁定的物料 (需要后续扩展)
- 不支持批量操作

### 7.2 建议优化
1. **智能分组排程** - 如果订单包含多种物料类型，自动分组
2. **解锁功能** - 允许用户取消已锁定的物料
3. **批量操作** - 支持批量锁定/提交多个订单
4. **审核流程** - 添加主管审核环节
5. **库存预警** - 锁定时提醒库存不足的物料
