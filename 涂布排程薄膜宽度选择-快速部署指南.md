## 第五步：数据验证 (5分钟)

### 验证锁定记录
```sql
-- 查看所有物料锁定记录
SELECT 
    sml.id,
    sml.schedule_id,
    sml.order_id,
    sml.locked_area,
    sml.lock_status,
    sml.locked_time,
    sc.task_no,
    sc.film_width
FROM schedule_material_lock sml
LEFT JOIN schedule_coating sc ON sml.schedule_id = sc.schedule_id
WHERE sml.lock_status = '锁定中'
ORDER BY sml.locked_time DESC;
```

### 验证库存变化
```sql
-- 查看薄膜库存的锁定情况
SELECT 
    material_code,
    material_name,
    width,
    thickness,
    total_area,
    available_area,
    locked_area,
    total_rolls,
    available_rolls,
    locked_rolls
FROM film_stock
WHERE locked_area > 0 OR locked_rolls > 0
ORDER BY width;
```

### 验证排程状态
```sql
-- 查看已锁定物料的排程
SELECT 
    task_no,
    order_no,
    material_code,
    film_width,
    material_lock_status,
    film_locked_time,
    plan_sqm
FROM schedule_coating
WHERE material_lock_status = 'locked'
ORDER BY film_locked_time DESC;
```

## 🚨 常见问题排查

### 问题1：点击"选择宽度"按钮无反应
**检查**：
1. F12打开浏览器控制台，查看是否有JavaScript错误
2. 检查Network标签，确认API请求是否发送
3. 确认后端服务是否正常运行

**解决**：
```bash
# 重启前端开发服务器
cd e:\vue\ERP
npm run dev
```

### 问题2：宽度列表为空
**检查**：
```sql
-- 检查是否有可用库存
SELECT * FROM film_stock WHERE available_area > 0;
```

**解决**：
如果没有数据，执行"第一步"中的测试数据插入SQL。

### 问题3：锁定失败
**检查后端日志**：
```
打开: e:\java\MES\logs\mes-application.log
查找: ERROR 关键字
```

**常见原因**：
1. 数据库字段未添加成功
2. 薄膜库存数据不存在
3. 订单信息不完整

### 问题4：API返回404
**检查**：
1. 后端是否重启
2. Controller类是否正确加载
3. 请求URL是否正确

**验证**：
```bash
# 测试API是否可访问
curl http://localhost:8080/api/film/available-widths
```
