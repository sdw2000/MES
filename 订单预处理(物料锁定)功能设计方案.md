# 订单预处理(物料锁定)功能完整设计方案

## 一、需求概述

### 目标
实现订单的人工预处理功能，通过物料选择和锁定，决定订单应该进行的生产步骤。

### 分离逻辑
- **已选择物料** → 直接进入 **复卷/分切待排池** (已有库存)
- **未选择物料** → 进入 **涂布待排池** (需要涂布生产)

### 3.1 新增表：订单预处理表 (order_preprocessing)

CREATE TABLE order_preprocessing (

    -- 订单信息

## 六、核心业务逻辑

### 6.1 自动判断排程类型

```java
/**
 * 根据是否选择库存自动判断排程类型
 */
private String determineScheduleType(OrderPreprocessing preprocessing) {
    Long lockedCount = orderMaterialLockMapper.countByPreprocessingId(preprocessing.getId());
    
    if (lockedCount > 0) {
        // 已选择物料，判断是复卷还是分切
        TapeStock tape = tapeStockMapper.selectById(lockedCount);
        
        if ("母卷".equals(tape.getRollType())) {
            return "复卷";  // 母卷需要复卷
        } else {
            return "分切";  // 复卷/分切卷直接分切
        }
    } else {
        return "涂布";      // 未选库存，需要涂布生产
    }
}
```

### 6.2 物料锁定机制 (乐观锁)

```java
/**
 * 锁定物料 (支持一个订单多次选择)
 */
@Transactional
public void lockMaterials(OrderMaterialLockRequest request) throws Exception {
    for (MaterialLockItem item : request.getLocks()) {
        TapeStock tape = tapeStockMapper.selectById(item.getTapeStockId());
        
        // 检查版本号 (乐观锁)
        if (!tape.getVersion().equals(item.getVersion())) {
            throw new Exception("物料库存已被其他订单修改，请重新选择");
        }
        
        // 更新 tape_stock 的 reserved_area
        tapeStockMapper.updateReservedArea(
            tape.getId(),
            tape.getReservedArea().add(item.getLockArea()),
            tape.getVersion(),
            tape.getVersion() + 1
        );
        
        // 插入锁定记录
        OrderMaterialLock lock = new OrderMaterialLock();
        lock.setOrderId(request.getOrderId());
        lock.setTapeStockId(item.getTapeStockId());
        lock.setLockArea(item.getLockArea());
        lock.setFifoOrder(item.getFifoOrder());
        orderMaterialLockMapper.insert(lock);
    }
}
```

### 6.3 FIFO 查询逻辑

```java
/**
 * 按 FIFO 顺序查询可锁定的物料
 */
public List<AvailableMaterialDTO> getAvailableMaterials(String materialCode) {
    return tapeStockMapper.selectAvailableMaterialsByCode(materialCode)
        .stream()
        .map(tape -> {
            int lockedRolls = orderMaterialLockMapper.countByTapeStockId(tape.getId());
            return new AvailableMaterialDTO()
                .setTapeStockId(tape.getId())
                .setQrCode(tape.getQrCode())
                .setBatchNo(tape.getBatchNo())
                .setTotalRolls(tape.getTotalRolls())
                .setLockedRolls(lockedRolls)
                .setAvailableRolls(tape.getTotalRolls() - lockedRolls)
                .setFifoOrder(tape.getSequenceNo());  // 按序列号排序
        })
        .sorted(Comparator.comparing(AvailableMaterialDTO::getFifoOrder))
        .limit(50)
        .collect(Collectors.toList());
}
```

## 八、关键设计要点

### 8.1 一物多单的处理
- 一卷 tape_stock 可被多个 OrderMaterialLock 记录引用
- 通过 `reserved_area` 记录总锁定面积
- 锁定时需要检查 `available_area` >= 锁定面积

### 8.2 FIFO 排序
- 使用 `sequence_no` 字段作为排序基准
- 查询时按 FIFO 顺序返回，让用户优先选择老库存

### 8.3 可编辑数量
- 订单明细数量可在此页面调整 (如无需全量)
- 调整后影响 schedule_type 的判断

### 8.4 路由分离
- 锁定物料 → `production_schedule` 或具体的排程表
- 未锁定物料 → `pending_coating_order_pool` (涂布待排池)

## 九、数据流图

```
销售订单
    ↓
订单预处理页面 (order_preprocessing)
    ↓
【是否有库存?】
    ├─ YES: 选择物料 → 记录到 order_material_lock
    │       ↓
    │   【自动判断】
    │   ├─ 母卷 → 复卷排程
    │   └─ 复卷/分切 → 分切排程
    │       ↓
    │   → production_schedule (生产排程表)
    │
    └─ NO: 空的锁定记录
            ↓
        → pending_coating_order_pool (涂布待排池)
```

## 十、测试场景

1. **场景1**: 订单有完整库存，一次性锁定
2. **场景2**: 订单库存分散，需要从多卷中锁定
3. **场景3**: 订单无库存，跳过物料选择，进入涂布池
4. **场景4**: 同一卷物料被多个订单锁定
5. **场景5**: 物料在锁定过程中被修改 (乐观锁冲突处理)
6. **场景6**: FIFO 查询返回顺序验证
