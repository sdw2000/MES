# 排程时间显示修复说明

## 问题
1. 前端涂布、复卷、分切任务表格中没有显示开始时间和结束时间
2. 后端涂布任务的开始时间计算有误（排程日期昨天时，开始时间不对）

## 修复内容

### 后端修复 - 时间计算逻辑
**文件**: [ProductionScheduleServiceImpl.java](e:\java\MES\src\main\java\com\fine\serviceIMPL\production\ProductionScheduleServiceImpl.java#L200-L220)

**问题原因**:
- 原逻辑先判断是否是今天，再设置08:00时间
- 导致排程日期是昨天时，使用的是昨天00:00而不是昨天08:00

**修复方案**:
```java
// 旧逻辑（有问题）
Calendar calendar = Calendar.getInstance();
calendar.setTime(scheduleDate);
boolean isToday = calendar.get(Calendar.YEAR) == today.get(Calendar.YEAR) &&
                 calendar.get(Calendar.DAY_OF_YEAR) == today.get(Calendar.DAY_OF_YEAR);
if (isToday && today.after(calendar)) {
    currentStartTime = new Date();
} else {
    calendar.set(Calendar.HOUR_OF_DAY, 8);
    currentStartTime = calendar.getTime();
}

// 新逻辑（修复后）
calendar.setTime(scheduleDate);
calendar.set(Calendar.HOUR_OF_DAY, 8);
calendar.set(Calendar.MINUTE, 0);
calendar.set(Calendar.SECOND, 0);
calendar.set(Calendar.MILLISECOND, 0);
Date scheduledStartTime = calendar.getTime();

Date now = new Date();
if (now.after(scheduledStartTime)) {
    currentStartTime = now;  // 排程时间已过，用当前时间
} else {
    currentStartTime = scheduledStartTime;  // 否则用排程日期08:00
}
```

**修复效果**:
- 排程日期未来 → 使用排程日期08:00
- 排程日期的08:00已过去 → 使用当前时间
- 时间计算准确，不会出现00:00的情况

#### 4. 时间格式化方法
**文件**: [schedule.vue](e:\vue\ERP\src\views\production\schedule.vue#L1502-L1511)

**新增方法**:
```javascript
formatDateTime(dateTime) {
  if (!dateTime) return '-'
  const d = new Date(dateTime)
  const year = d.getFullYear()
  const month = String(d.getMonth() + 1).padStart(2, '0')
  const day = String(d.getDate()).padStart(2, '0')
  const hour = String(d.getHours()).padStart(2, '0')
  const minute = String(d.getMinutes()).padStart(2, '0')
  return `${month}-${day} ${hour}:${minute}`
}
```

**显示格式**: `01-15 08:00` (月-日 时:分)
