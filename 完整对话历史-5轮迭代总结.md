```java
    // 未被选中订单也被减少

```

**正确代码**：
```java
// ✅ 正确：只迭代scheduleQtyMap中的订单
if (scheduleQtyMap != null && !scheduleQtyMap.isEmpty()) {
    for (Map.Entry<Long, Integer> entry : scheduleQtyMap.entrySet()) {
        Long orderItemId = entry.getKey();
        Integer scheduleQty = entry.getValue();
        if (scheduleQty != null && scheduleQty > 0) {
            salesOrderItemMapper.decreaseRolls(orderItemId, scheduleQty);
        }
    }
}
```

```javascript
async loadPendingOrders() {
    const res = await getPendingOrderItems(this.pendingParams);
    if (res.code === 200) {
        // 使用对象解构确保schedule_qty属性存在
        const processedList = (res.data.list || []).map((order) => {
            return {
                ...order,
                schedule_qty: order.schedule_qty || order.pending_qty || 0
            };
        });
        this.pendingOrders = processedList;
    }
}
```

### 第5轮 - 涂布面积(planSqm)计算逻辑改进 ⭐ **新完成**

| 项目 | 详情 |
|------|------|
| **问题** | 涂布面积计算逻辑不明确，硬编码值不符实际 |
| **根本原因** | 涂布长度固定5000米，母卷宽度固定1200mm |
| **修复文件** | `ProductionScheduleServiceImpl.java` (408-640行) |
| **解决方案** | 实现动态计算，根据订单数据计算 |
| **验收状态** | ✅ 完成 |
| **编译状态** | ✅ BUILD SUCCESS |
| **文档** | 4份详细文档 |

**计算公式**：
```
planSqm(m²) = (涂布长度(m) × 母卷宽度(mm)) / 1,000,000

涂布长度 = Σ(订单长度 × 订单数量)  ← 动态计算
母卷宽度 = MAX(订单宽度)  ← 从订单取值
```

**新增方法**：
- `calculateTotalLength()` - 计算总涂布长度
- `determineJumboWidth()` - 确定母卷宽度

**改进内容**：
- ✅ 涂布长度从固定改为动态
- ✅ 母卷宽度从固定改为动态
- ✅ 添加完善的日志记录
- ✅ 添加详细的异常处理
- ✅ 生成4份详细文档
