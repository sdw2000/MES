## 【常见问题排查】

### 问题1: 分切任务页面显示404

**原因:** 前端路由没有正确加载
**解决:**
```
1. 检查 e:\vue\ERP\src\views\production\slittingTasks.vue 文件内容
   应该引入: <slitting-tasks-new />
   
2. 清除浏览器缓存:
   按 Ctrl+Shift+Delete 打开清除缓存对话框
   
3. 重启前端服务:
   npm run dev
```

### 问题2: 新增任务后,表格显示为空

**原因:** 前端API调用路径错误
**解决:**
```
1. 打开浏览器开发者工具 (F12)
2. 切换到 Network 标签
3. 点击新增,检查请求URL
   应该是: /api/production/schedule/slitting/add
   
4. 查看响应:
   应该返回成功响应,包含新建任务的ID
```

### 问题3: 时间显示为NaN或Invalid Date

**原因:** 时间戳格式不匹配
**解决:**
```
1. 检查后端返回的时间格式
   应该是: "2024-12-20 14:20:00"
   
2. 修改前端时间显示格式
   文件: SlittingTasksNew.vue
   方法: formatTime()
```

### 问题4: 时间精度没有生效

**原因:** 前端没有应用时间四舍五入逻辑
**解决:**
```
确认: 这是后端应用的逻辑,不是前端逻辑
1. 输入: 2024-12-20 14:23:00
2. 后端处理: (23+5)/10*10 = 20
3. 保存到数据库: 2024-12-20 14:20:00
4. 前端显示: 2024-12-20 14:20

如果显示为14:23,说明后端没有应用时间精度处理
检查: ProductionScheduleServiceImpl.java 中 roundTimeToTenMinutes() 方法是否被调用
```
