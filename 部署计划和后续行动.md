## ⚠️ 回滚方案

### 快速回滚（如出现严重问题）

**触发条件**：
- 错误率 > 5%
- API响应时间 > 1000ms
- 业务数据不一致

**回滚步骤**：
```bash
# 1. 停止应用
systemctl stop mes-service

# 2. 恢复旧版本
cp /backups/mes-0.9.0.jar /opt/mes/
systemctl start mes-service

# 3. 验证
curl http://localhost:8080/api/production/schedule/list

# 4. 恢复数据库（如需要）
mysql < /backups/database-backup.sql
```

**回滚时间**：< 15分钟

**沟通流程**：
1. 技术团队发现问题
2. 向管理层报告
3. 执行回滚决定
4. 执行回滚操作
5. 验证回滚成功
6. 通知相关部门
