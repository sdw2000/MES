# 分切任务功能完整实现方案

## 【项目概述】

根据用户需求,对MES/ERP系统中的分切任务管理进行全面升级。本方案实现了一个完整的、面向生产的分切任务管理系统,支持:

- 11个关键业务字段的显示和编辑
- 10分钟精度的时间排程
- 订单完整追踪
- 操作人员责任追踪
- 完整的CRUD操作界面

---

## 【核心需求分析】

### 用户需求
```
分切任务表需要显示以下11个字段:
1. ✅ 分切任务号 - 自动生成(SLT-YYYYMMDD-XXXXX)
2. ✅ 订单号 - 来自关联订单
3. ✅ 详情号 - 订单明细号
4. ✅ 规格 - 综合显示(材料-宽度-长度-卷数)
5. ✅ 数量 - 计划卷数
6. ✅ 机台号 - 设备编号
7. ✅ 计划时间 - 精确到10分钟(开始和结束)
8. ✅ 实际开始时间 - 生产开始时间(可编辑)
9. ✅ 操作人 - 执行分切的员工
10. ✅ 状态 - 任务状态(待生产/进行中/已完成)
11. ✅ 备注 - 任务说明信息
```

### 时间精度要求
```
所有排程时间必须精确到10分钟:
- 14:23 → 14:20
- 14:26 → 14:30
- 15:55 → 16:00
- 实现在服务层的 roundTimeToTenMinutes() 方法
```

```java
@Override
public IPage<ScheduleSlitting> getSlittingTasks(Map<String, Object> params) {
    // 1. 提取分页参数
    Integer pageNum = (Integer) params.get("pageNum");    // 默认1
    Integer pageSize = (Integer) params.get("pageSize");  // 默认20
    
    // 2. 构建查询条件
    Page<ScheduleSlitting> page = new Page<>(pageNum, pageSize);
    QueryWrapper<ScheduleSlitting> wrapper = new QueryWrapper<>();
    
    // 3. 支持的查询条件
    if (!StringUtils.isEmpty(orderNo)) {
        wrapper.like("order_no", orderNo);        // 订单号模糊查询
    }
    if (!StringUtils.isEmpty(status)) {
        wrapper.eq("status", status);              // 状态精确查询
    }
    if (!StringUtils.isEmpty(planDate)) {
        wrapper.like("plan_start_time", planDate); // 计划日期范围
    }
    if (equipmentId != null) {
        wrapper.eq("equipment_id", equipmentId);   // 设备精确查询
    }
    
    // 4. 按创建时间倒序
    wrapper.orderByDesc("create_time");
    
    // 5. 返回分页结果
    return slittingMapper.selectPage(page, wrapper);
}
```

**关键方法 - addSlittingTask**
```java
@Override
@Transactional
public ScheduleSlitting addSlittingTask(ScheduleSlitting slitting) {
    // 1. 参数校验
    if (slitting == null || StringUtils.isEmpty(slitting.getOrderNo())) {
        throw new RuntimeException("订单号不能为空");
    }
    
    // 2. 生成任务号(格式: SLT-YYYYMMDD-XXXXX)
    if (StringUtils.isEmpty(slitting.getTaskNo())) {
        slitting.setTaskNo(generateSlittingTaskNo());
    }
    
    // 3. 应用10分钟时间精度
    slitting.setPlanStartTime(roundTimeToTenMinutes(slitting.getPlanStartTime()));
    slitting.setPlanEndTime(roundTimeToTenMinutes(slitting.getPlanEndTime()));
    
    // 4. 自动生成规格字符串(如: FT-001-50mm x 100m x 100卷)
    if (StringUtils.isEmpty(slitting.getSpec())) {
        String spec = String.format("%s-%s x %s x %d卷",
            slitting.getMaterialCode(),
            slitting.getTargetWidth(),
            slitting.getSlitLength(),
            slitting.getPlanRolls() != null ? slitting.getPlanRolls() : 0);
        slitting.setSpec(spec);
    }
    
    // 5. 设置默认值
    slitting.setCreateTime(new Date());
    slitting.setUpdateTime(new Date());
    slitting.setStatus("pending");
    
    // 6. 插入数据库
    slittingMapper.insert(slitting);
    return slitting;
}
```

**关键工具方法 - roundTimeToTenMinutes**
```java
private Date roundTimeToTenMinutes(Date date) {
    if (date == null) {
        return new Date();
    }
    
    Calendar cal = Calendar.getInstance();
    cal.setTime(date);
    
    // 取得当前分钟数
    int minute = cal.get(Calendar.MINUTE);
    
    // 四舍五入到10的倍数
    // 例: 23分 → (23+5)/10*10 = 20分
    //     26分 → (26+5)/10*10 = 30分
    int roundedMinute = (minute + 5) / 10 * 10;
    
    // 处理60分钟边界(向上进位到下一小时)
    if (roundedMinute >= 60) {
        cal.add(Calendar.HOUR_OF_DAY, 1);
        roundedMinute = 0;
    }
    
    cal.set(Calendar.MINUTE, roundedMinute);
    cal.set(Calendar.SECOND, 0);
    cal.set(Calendar.MILLISECOND, 0);
    
    return cal.getTime();
}
```

## 【关键业务逻辑】

### 1. 时间精度处理流程
```
前端输入: 2024-12-20 14:23:00
     ↓
后端接收 (ProductionScheduleController)
     ↓
调用 roundTimeToTenMinutes()
     ↓
处理逻辑:
  - 分钟: (23 + 5) / 10 * 10 = 20
  - 秒/毫秒: 清零
     ↓
处理结果: 2024-12-20 14:20:00
     ↓
保存到数据库
     ↓
前端显示: 14:20
```

### 2. 任务号生成流程
```
触发: 新增任务,任务号为空
     ↓
调用 generateSlittingTaskNo()
     ↓
获取当前日期: 2024-12-20 → "20241220"
     ↓
查询该日期最大序列号: SELECT MAX(sequence)
     ↓
递增序列: 1 → 2 → 3...
     ↓
组合任务号: SLT-20241220-00001
     ↓
返回生成的任务号
```

### 3. 规格自动生成流程
```
新增任务时收集信息:
  - materialCode: "FT-001"
  - targetWidth: 50
  - slitLength: 100
  - planRolls: 100
     ↓
调用规格生成逻辑:
  String spec = String.format("%s-%smm x %sm x %d卷",
    materialCode,
    targetWidth,
    slitLength,
    planRolls)
     ↓
生成结果: "FT-001-50mm x 100m x 100卷"
     ↓
自动填充到 spec 字段
```

### 4. 状态流转流程
```
新增任务
    ↓
status = "pending" (待生产)
    ↓ (用户点击"开始"按钮)
    ↓
status = "in_progress" (进行中)
actualStartTime = 当前时间
    ↓ (用户点击"完成"按钮)
    ↓
status = "completed" (已完成)
actualEndTime = 当前时间
    ↓
任务完成
```
