# 🔧 排程参数类型转换修复说明

## 问题描述

**错误信息**：
```
{"code":500,"msg":"智能排程失败: java.lang.String cannot be cast to java.lang.Number"}
```

**根本原因**：
前端发送的 `order_item_id` 和 `schedule_qty` 是字符串类型（JSON序列化），但后端代码直接将其强制转换为 `Number` 类型，导致类型转换失败。

### 前端发送的数据格式
```javascript
{
  scheduleDate: "2026-01-10",
  scheduleType: "order",
  details: [
    {order_item_id: "2693", schedule_qty: 100},
    {order_item_id: "3006", schedule_qty: 20}
  ],
  operator: "admin"
}
```

注意：`order_item_id` 是字符串 `"2693"`，不是数字。

---

## 修复方案

**文件**：`ProductionScheduleController.java` (第149-205行)

### 修复内容

#### 1️⃣ 为 `order_item_id` 添加类型判断
```java
// 处理order_item_id：可能是String或Number
Object orderItemIdObj = detailMap.get("order_item_id");
Long orderItemId;
if (orderItemIdObj instanceof String) {
    orderItemId = Long.parseLong((String) orderItemIdObj);
} else if (orderItemIdObj instanceof Number) {
    orderItemId = ((Number) orderItemIdObj).longValue();
} else {
    continue;  // 跳过无效的值
}
```

#### 2️⃣ 为 `schedule_qty` 添加类型判断
```java
// 处理schedule_qty：可能是String或Number
Object scheduleQtyObj = detailMap.get("schedule_qty");
Integer scheduleQty;
if (scheduleQtyObj instanceof String) {
    scheduleQty = Integer.parseInt((String) scheduleQtyObj);
} else if (scheduleQtyObj instanceof Number) {
    scheduleQty = ((Number) scheduleQtyObj).intValue();
} else {
    continue;  // 跳过无效的值
}
```

#### 3️⃣ 修复兼容旧参数的代码
```java
// 支持String和Number两种类型
orderItemIds = rawIds.stream()
    .map(id -> {
        if (id instanceof String) {
            return Long.parseLong((String) id);
        } else if (id instanceof Number) {
            return ((Number) id).longValue();
        }
        return null;
    })
    .filter(id -> id != null)
    .collect(java.util.stream.Collectors.toList());
```

---

## 修复要点

| 问题 | 原代码 | 修复后 |
|------|--------|--------|
| 类型转换失败 | `((Number) obj).longValue()` | 先判断是String还是Number，再转换 |
| 无效数据处理 | 直接异常 | 使用continue跳过，或filter过滤 |
| 兼容性 | 只支持Number | 同时支持String和Number |
| 错误定位 | 笼统的异常消息 | 在转换前检查，更清晰 |
