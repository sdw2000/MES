## 🎨 本轮（第5轮）- 涂布面积计算逻辑

### 问题描述

**原问题**：涂布计划中的`planSqm`（计划涂布面积）计算逻辑不明确

**症状**：
- 涂布长度硬编码为5000米
- 母卷宽度硬编码为1200mm
- 计算结果不符合实际业务需求

### 解决方案

#### 📐 计算公式确认

```
planSqm(m²) = (涂布长度(m) × 母卷宽度(mm)) / 1,000,000
```

**数据来源**：
- **涂布长度**：根据订单动态计算 = Σ(订单长度 × 订单数量)
- **母卷宽度**：从订单中取最大值，默认1200mm
- **平方米**：两者相乘再除以1000000

#### 🔧 代码改进

**改进1：动态计算涂布长度**

新增方法：`calculateTotalLength(List<Map<String, Object>> items)`

```java
private BigDecimal calculateTotalLength(List<Map<String, Object>> items) {
    BigDecimal totalLength = BigDecimal.ZERO;
    
    for (Map<String, Object> item : items) {
        // 获取长度（毫米）
        BigDecimal length = parseLength(item.get("length"));
        
        // 获取数量（卷数）
        Integer rolls = parseRolls(item.get("rolls"));
        
        // 累加：length(mm) × rolls
        totalLength = totalLength.add(length.multiply(new BigDecimal(rolls)));
    }
    
    return totalLength;
}
```

**特点**：
- ✅ 支持多种数据类型（String、Number、BigDecimal）
- ✅ 完善异常处理
- ✅ 详细日志记录
- ✅ 向后兼容

**改进2：动态确定母卷宽度**

新增方法：`determineJumboWidth(List<Map<String, Object>> items)`

```java
private Integer determineJumboWidth(List<Map<String, Object>> items) {
    Integer maxWidth = 0;
    
    // 从订单中找最大宽度
    for (Map<String, Object> item : items) {
        Integer width = parseWidth(item.get("width"));
        if (width > maxWidth) {
            maxWidth = width;
        }
    }
    
    // 默认值1200mm
    return maxWidth > 0 ? maxWidth : 1200;
}
```

**特点**：
- ✅ 灵活适配不同订单宽度
- ✅ 自动降级到默认值
- ✅ 详细日志记录

## 💡 关键知识点

### 1. 涂布面积计算三步法
```
第1步：计算总长度
  = Σ(订单长度 × 订单数量)

第2步：确定母卷宽度
  = MAX(订单宽度) 或 默认1200mm

第3步：计算平方米
  = (总长度 × 母卷宽度) / 1,000,000
```

### 2. 类型转换安全模式
```java
// 判断类型
if (obj instanceof String) {

    // 处理未知类型
}
```

### 3. BigDecimal精确计算
```java
// 避免浮点精度问题
BigDecimal result = new BigDecimal(5000)
    .multiply(new BigDecimal(1200))
    .divide(new BigDecimal(1000000), 2, BigDecimal.ROUND_HALF_UP);
```
