# 订单号和计划面积显示修复说明

## 问题现象
- ✅ 膜宽(mm) 显示正常（1040）
- ❌ 订单号 显示 "-"
- ❌ 计划面积 未显示

## 根本原因

### 1. 订单号字段缺失
- **Entity问题**：`ScheduleCoating.orderNo` 被错误标记为 `@TableField(exist=false)`
- **数据库问题**：`schedule_coating` 表缺少 `order_no`、`order_id`、`order_item_id` 字段
- **业务逻辑问题**：锁定物料时未设置订单信息

### 2. 计划面积字段映射
- `plan_sqm` 在数据库中存在，但锁定物料时计算的值未持久化到数据库
- 需要在锁定物料时从 `plan_length * jumbo_width / 1000000` 计算

```java
// 查询关联的订单信息
QueryWrapper<ScheduleOrderItem> orderWrapper = new QueryWrapper<>();
orderWrapper.eq("schedule_id", lockDTO.getScheduleId());
orderWrapper.last("LIMIT 1");
ScheduleOrderItem orderItem = orderItemMapper.selectOne(orderWrapper);

// 设置订单信息（如果查询到）
if (orderItem != null) {
    schedule.setOrderNo(orderItem.getOrderNo());
    schedule.setOrderId(orderItem.getOrderId());
    schedule.setOrderItemId(orderItem.getOrderItemId());
}

// 计算并设置计划面积
if (schedule.getPlanLength() != null && selectedStock.getWidth() != null) {
    BigDecimal planSqm = schedule.getPlanLength()
        .multiply(new BigDecimal(selectedStock.getWidth()))
        .divide(new BigDecimal(1000000), 2, BigDecimal.ROUND_HALF_UP);
    schedule.setPlanSqm(planSqm);
}
```

## 技术细节

### 字段关系

```
ProductionSchedule (id=26)
    ↓ (schedule_id)
ScheduleOrderItem (order_no, order_id, order_item_id)
    ↓ (通过查询关联)
ScheduleCoating (id=324, 325)
    - order_no: 从ScheduleOrderItem获取
    - order_id: 从ScheduleOrderItem获取
    - order_item_id: 从ScheduleOrderItem获取
    - jumbo_width: 从FilmStock.width获取 (1040)
    - plan_sqm: 计算 = plan_length * jumbo_width / 1000000
```

### 数据流程

1. **锁定物料时**：
   - 选择薄膜库存（宽度1040mm）
   - 查询schedule_order_item获取订单信息
   - 更新schedule_coating：
     * `jumbo_width` = 1040
     * `order_no` = "1011-R02-12..."
     * `order_id` = xxx
     * `order_item_id` = xxx
     * `plan_sqm` = 计算值

2. **查询详情时**：
   - MyBatis查询schedule_coating表
   - 自动映射：orderNo, planSqm, jumboWidth
   - 手动同步：filmWidth = jumboWidth（前端显示字段）

## 注意事项

1. **MySQL版本兼容性**：使用 `ADD COLUMN IF NOT EXISTS` 语法（MySQL 5.7+）
2. **事务安全**：锁定物料操作使用 `@Transactional` 保证原子性
3. **字段同步**：filmWidth 是临时字段，每次查询时同步自 jumboWidth
4. **订单关联**：一个排程可能有多个订单，取第一个（`LIMIT 1`）
