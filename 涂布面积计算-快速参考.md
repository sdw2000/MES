# 🎯 涂布面积(planSqm)计算 - 快速参考

## 📋 一句话总结
**涂布面积 = (订单总长度 × 母卷宽度) / 1,000,000**

---

## 📐 计算公式

```
planSqm(m²) = (coatingLength(m) × jumboWidth(mm)) / 1,000,000
```

### 单位说明
| 字段 | 单位 | 说明 |
|------|------|------|
| `coatingLength` | 米(m) | 涂布长度（动态计算） |
| `jumboWidth` | 毫米(mm) | 母卷宽度（从订单取值） |
| `planSqm` | 平方米(m²) | 计划面积（最终结果） |

---

## 🔢 计算示例

### 简单示例
```
输入：
  - 涂布长度：5000 m
  - 母卷宽度：1200 mm

计算：
  planSqm = (5000 × 1200) / 1,000,000
          = 6,000,000 / 1,000,000
          = 6.00 m²
```

### 复杂示例
```
输入订单：
  订单1：长度1000m, 宽度1200mm, 数量100卷
  订单2：长度2000m, 宽度1200mm, 数量50卷

第1步：计算总长度
  = (1000 × 100) + (2000 × 50)
  = 100,000 + 100,000
  = 200,000 m

第2步：确定最大宽度
  = max(1200, 1200)
  = 1200 mm

第3步：计算面积
  = (200,000 × 1200) / 1,000,000
  = 240.00 m²
```

## 📍 数据来源

### 涂布长度（coatingLength）
```
来源：订单项目表(sales_order_items)
计算：Σ(订单长度 × 订单数量)

示例：
  SELECT 
    SUM(length * rolls) as total_length
  FROM sales_order_items
  WHERE material_code = 'A001'
    AND status = 'pending'
```

### 母卷宽度（jumboWidth）
```
来源：订单项目表(sales_order_items)
选择：MAX(订单宽度)

示例：
  SELECT MAX(width) as max_width
  FROM sales_order_items
  WHERE material_code = 'A001'
    AND status = 'pending'
```

## 🛠️ 代码位置

### Java实现

| 文件 | 方法 | 说明 |
|------|------|------|
| `ProductionScheduleServiceImpl.java` | `generateCoatingTasks()` | 主方法 |
| `ProductionScheduleServiceImpl.java` | `calculateTotalLength()` | 计算总长度 |
| `ProductionScheduleServiceImpl.java` | `determineJumboWidth()` | 确定母卷宽度 |

### 关键代码
```java
// 第一步：计算总长度
BigDecimal totalLength = calculateTotalLength(items);

// 第二步：转换为米
int coatingLengthMeters = totalLength.divide(new BigDecimal(1000), 0, BigDecimal.ROUND_HALF_UP).intValue();

// 第三步：确定宽度
Integer jumboWidth = determineJumboWidth(items);

// 第四步：计算平方米
BigDecimal planSqm = new BigDecimal(coatingLengthMeters)
    .multiply(new BigDecimal(jumboWidth))
    .divide(new BigDecimal(1000000), 2, BigDecimal.ROUND_HALF_UP);
```

## 📊 常见场景

### 场景1：单料号单宽度
```
✓ 最简单的场景
- 所有订单：同一料号，同一宽度
- 计算方式：简单求和

示例结果：
  涂布长度：100,000 m
  母卷宽度：1200 mm
  面积：120 m²
```

### 场景2：单料号多宽度
```
✓ 常见场景
- 同一料号，多个宽度
- 使用最大宽度
- 若有多个涂布任务，按宽度分组（可选优化）

示例结果：
  涂布长度：50,000 m
  母卷宽度：1200 mm（取最大）
  面积：60 m²
```

### 场景3：多料号
```
✓ 正常场景
- 不同料号按料号分组
- 为每个料号生成一个涂布任务
- 分别计算其长度和面积

结果列表：
  料号A001: 涂布长度100,000m, 面积120m²
  料号A002: 涂布长度50,000m, 面积60m²
```

## 🔧 常见问题

### Q1: 平方米结果为0
**A:**
```
原因1：涂布长度为0
  → 检查订单长度是否有效

原因2：母卷宽度为0
  → 检查订单宽度是否有效

解决：
  1. 查看日志找出问题订单
  2. 检查订单明细表数据
  3. 重新创建排程
```

### Q2: 面积不准确
**A:**
```
原因1：数据类型转换错误
  → 查看WARN日志

原因2：精度丢失
  → 确保使用BigDecimal计算

原因3：单位混淆
  → 确认length单位为毫米mm

解决：
  1. 启用DEBUG日志
  2. 追踪每一步计算
  3. 验证中间值
```

### Q3: 设备不匹配
**A:**
```
原因：设备不可用或无涂布设备

解决：
  1. 检查设备表数据
  2. 确保设备状态为可用
  3. 确保device_type='COATING'
  4. 增加涂布设备
```
