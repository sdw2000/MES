# 手动排程功能 - 完整实现文档

## 📋 功能概述

手动排程系统允许用户手动管理订单排程，支持库存匹配锁定、涂布需求计算、复卷和分切排程的全流程管理。

## 🎯 核心功能

### 1. 订单排程主页
- **订单列表展示**：显示所有待排程订单（订单数量 > 已排数量）
- **可编辑排程数量**：用户可调整本次排程数量
- **库存选择锁定**：点击"选择库存"按钮，系统自动匹配库存（先进先出排序）
- **涂布需求计算**：点击"计算涂布"按钮，自动聚合同料号前缀的所有订单需求

### 2. 涂布排程标签页
- **涂布任务列表**：显示需要涂布的订单
- **机台分配**：可选择涂布机台（涂布机1/2/3号）
- **时间安排**：可自定义涂布排程日期时间
- **状态跟踪**：待排程 → 涂布已排 → 复卷已排 → 已完成

## 📊 数据流程

```
订单查询
    ↓
输入排程数量
    ↓
选择库存 ──→ 库存充足 ──→ 锁定库存 ──→ 创建复卷排程
    ↓           ↓
    └──→ 库存不足 ──→ 计算涂布需求 ──→ 涂布排程 ──→ 复卷排程
```

## 🔑 核心功能详解

### 1. 待排程订单查询
**SQL逻辑**：
```sql
SELECT 
  o.order_number AS order_no,
  od.id AS order_detail_id,
  od.material_code,
  od.material_name,
  od.width, od.length, od.thickness,
  od.roll_count AS order_qty,
  IFNULL(SUM(ms.schedule_qty), 0) AS scheduled_qty,
  (od.roll_count - IFNULL(SUM(ms.schedule_qty), 0)) AS remaining_qty
FROM order_details od
JOIN orders o ON od.order_id = o.id
LEFT JOIN manual_schedule ms ON od.id = ms.order_detail_id
WHERE o.status != 'COMPLETED'
GROUP BY od.id
HAVING remaining_qty > 0
ORDER BY o.delivery_date ASC
```

### 2. 库存匹配（先进先出）
**算法**：
- 按料号前缀匹配（FT-001-38mm 匹配 FT-001 开头的库存）
- 按宽度、厚度精确匹配
- 按生产日期升序排序（先进先出）
- 计算可用总量，判断是否充足

**示例**：
```javascript
需求: FT-001-38mm, 100卷
库存:
  ✓ FT-001-母卷, 2025-01-10, 50卷  (优先)
  ✓ FT-001-复卷, 2025-01-15, 60卷
  
总可用: 110卷 ≥ 100卷 → 库存充足
```

### 3. 涂布需求计算
**聚合逻辑**：
- 提取料号前缀（FT-001-38mm → FT）
- 查询当前订单交期及以后的所有相同前缀订单
- 按厚度分组，计算总需求卷数和面积

**计算公式**：
```
涂布面积(㎡) = Σ (宽度mm ÷ 1000 × 长度m × 需求卷数)
```

**示例**：
```javascript
订单A: FT-001-38mm, 交期6/27, 50卷
订单B: FT-002-38mm, 交期6/30, 30卷  (相同前缀FT, 交期更晚)

聚合结果:
  料号前缀: FT
  厚度: 100μm
  总需求: 80卷
  涂布面积: (38÷1000 × 1000 × 50) + (38÷1000 × 1000 × 30) = 3040㎡
```

### 4. 状态流转
```
PENDING (待排程)
    ↓
COATING_SCHEDULED (涂布已排) ──→ 库存不足，需要涂布
    ↓
REWINDING_SCHEDULED (复卷已排) ──→ 涂布完成或库存充足
    ↓
COMPLETED (已完成)
```

- **已排数量**：已经排程的卷数
- **剩余数量**：还需排程的卷数
- **本次排程数量**：用户输入框，可调整

**操作按钮**：
1. **选择库存**：弹出库存匹配对话框
   - 显示先进先出排序的库存列表
   - 用户手动分配每条库存的使用数量
   - 确认后锁定库存并创建复卷排程

2. **计算涂布**：弹出涂布需求计算对话框
   - 自动聚合同料号前缀的所有订单
   - 显示计算出的涂布面积（可手动调整）
   - 确认后保存到涂布排程标签页

### Tab 2: 涂布排程
**字段说明**：
- **订单号/产品编码/名称**：订单信息
- **涂布面积(㎡)**：需要涂布的平米数
- **计划时间**：日期时间选择器（用户手动选择）
- **机台**：下拉选择涂布机1/2/3号
- **状态**：当前排程状态

**操作**：
- 点击"确认排程"后，将涂布任务正式排入机台计划

## 📌 注意事项

### 1. 库存锁定机制
- 使用乐观锁防止超卖（TapeStock.version字段）
- 锁定后库存立即扣减，不可撤销
- 建议先确认数量再操作

### 2. 涂布需求聚合
- 只聚合**当前订单交期及以后**的订单
- 只聚合**相同料号前缀**的订单（FT-001 和 FT-002 都属于FT前缀）
- 按**厚度**分组（不同厚度需要分别涂布）

### 3. 先进先出（FIFO）
- 系统自动按生产日期升序排序
- 用户分配时应优先使用排在前面的库存
- 避免库存积压

### 4. 数据一致性
- 所有操作使用事务（@Transactional）
- 库存锁定失败会回滚整个排程
- 建议在业务低峰期进行批量排程
