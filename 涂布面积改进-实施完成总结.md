## 🔍 问题分析

### 原问题
在 `ProductionScheduleServiceImpl.java` 的 `generateCoatingTasks()` 方法中：
- **涂布长度** 固定为 5000 米（硬编码）
- **母卷宽度** 固定为 1200 mm（硬编码）
- 两个值都与实际订单数据无关，导致计算出的涂布面积不准确

```java
// ❌ 旧代码
Integer coatingLength = 5000;        // 固定值
BigDecimal planSqm = new BigDecimal(coatingLength)
    .multiply(new BigDecimal(1200))  // 固定值
    .divide(new BigDecimal(1000000), 2, BigDecimal.ROUND_HALF_UP);
```

### 计算公式
```
planSqm(平方米) = (涂布长度米 × 母卷宽度毫米) / 1,000,000
```

## ✅ 实施方案

### 改进1：动态计算涂布长度

**方法**：`calculateTotalLength(List<Map<String, Object>> items)`

**逻辑**：
```
总涂布长度(毫米) = Σ(订单长度 × 订单数量)

示例：
  订单1：1000m × 100卷 = 100,000m
  订单2：2000m × 50卷  = 100,000m
  ────────────────────────────────
  总计：200,000m = 200,000,000mm
```

**代码实现**：
```java
private BigDecimal calculateTotalLength(List<Map<String, Object>> items) {
    BigDecimal totalLength = BigDecimal.ZERO;
    
    for (Map<String, Object> item : items) {
        // 获取长度（毫米）- 支持String、Number、BigDecimal三种类型
        BigDecimal length = parseLength(item.get("length"));
        
        // 获取数量（卷数）
        Integer rolls = parseRolls(item.get("rolls"));
        
        // 累加：length(mm) × rolls
        totalLength = totalLength.add(length.multiply(new BigDecimal(rolls)));
    }
    
    return totalLength;
}
```

**特点**：
- ✅ 支持多种数据类型（String、Number、BigDecimal）
- ✅ 完善的异常处理
- ✅ 详细的日志记录
- ✅ 向后兼容性

### 改进2：动态确定母卷宽度

**方法**：`determineJumboWidth(List<Map<String, Object>> items)`

**优先级**：
1. **首选**：使用订单中的最大宽度
2. **备选**：使用系统默认值 1200mm

**代码实现**：
```java
private Integer determineJumboWidth(List<Map<String, Object>> items) {
    Integer maxWidth = 0;
    
    // 从订单中找最大宽度
    for (Map<String, Object> item : items) {
        Integer width = parseWidth(item.get("width"));
        if (width > maxWidth) {
            maxWidth = width;
        }
    }
    
    // 如果没有有效宽度，使用默认值
    return maxWidth > 0 ? maxWidth : 1200;
}
```

**特点**：
- ✅ 灵活适配不同订单宽度
- ✅ 自动降级到默认值
- ✅ 详细的日志记录

## 📝 详细计算流程

### 场景示例

```
【订单信息】
订单1（PO-001）：长度=1000m, 宽度=1200mm, 数量=100卷
订单2（PO-002）：长度=2000m, 宽度=1200mm, 数量=50卷


【处理过程】

第1步：分组
  同料号订单分在一组 → [PO-001, PO-002]

第2步：计算总长度
  total_length = (1000m × 100卷) + (2000m × 50卷)
               = 1000000m + 100000m
               = 1100000m
               = 1,100,000,000mm

第3步：转换为米
  coatingLength = 1,100,000,000mm ÷ 1000 = 1,100,000 m

第4步：确定宽度
  max_width = max(1200, 1200) = 1200mm

第5步：计算平方米
  planSqm = (1,100,000 × 1200) / 1,000,000
          = 1,320,000,000 / 1,000,000
          = 1,320 m²

【输出】
涂布任务创建成功
├─ 涂布长度：1,100,000 m
├─ 母卷宽度：1200 mm
└─ 计划面积：1320.00 m²
```

## 🔍 故障排查

### 问题1：计算结果为0
**原因**：订单长度为0或数量为0  
**解决**：
- 检查订单数据是否完整
- 确保length和rolls字段有有效值

### 问题2：宽度显示错误
**原因**：订单宽度数据类型转换失败  
**解决**：
- 查看日志中的WARN信息
- 检查width字段的数据类型

### 问题3：平方米计算精度
**原因**：BigDecimal精度设置  
**解决**：
- 使用 `BigDecimal.ROUND_HALF_UP` 进行四舍五入
- 设置2位小数精度
