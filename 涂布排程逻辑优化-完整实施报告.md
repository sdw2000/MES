**视图核心逻辑：**
```sql
SELECT 
    soi.id AS order_item_id,
    so.id AS order_id,
    so.order_no,
    soi.material_code,
    soi.material_name,
    (soi.quantity - IFNULL(soi.scheduled_qty, 0)) AS pending_qty,
    (soi.quantity - IFNULL(soi.scheduled_qty, 0)) * soi.width * soi.length / 1000000 AS pending_area,
    mpc.min_production_area  -- MOQ检查
FROM sales_order_items soi
INNER JOIN sales_orders so ON soi.order_id = so.id
LEFT JOIN material_production_config mpc ON soi.material_code = mpc.material_code
WHERE soi.quantity > IFNULL(soi.scheduled_qty, 0)
```

2. **getPendingOrdersGroupByMaterial()** - 按物料分组统计
   - 返回每个物料的待排程总量、面积
   - 按交货日期和面积排序

   - 返回该物料的所有待排程订单明细

4. **autoScheduleCoating(materialCode, filmWidth, operator)** ★核心方法
   ```java
   逻辑流程：
   1. 查询该物料的所有待排程订单
   2. 查询物料的生产配置（MOQ）
   3. 计算总需求面积
   4. MOQ检查（警告但继续）
   5. 为每个订单创建涂布任务：
      - 生成任务单号：CT-YYYYMMDD-XXX
      - 关联订单：orderItemId, orderId, orderNo
      - 设置产品信息：materialCode, materialName
      - 设置计划面积：planSqm = pending_area
      - 设置薄膜宽度：filmWidth
      - 计算工时：setupTime + (area × unitTime)
      - 状态：pending
   6. 插入数据库
   7. 更新订单已排程数量：scheduled_qty += pending_qty
   ```

5. **batchScheduleCoating(orderItemIds, filmWidth, planDate, operator)**
   - 批量选择订单进行排程
   - 自动计算待排程数量
   - 批量创建涂布任务

## 🎯 核心实现原理

### 涂布排程自动化流程

```
销售订单 → 待排程订单视图 → 自动排程 → 涂布任务 → 更新已排程数量
   ↓              ↓                ↓             ↓             ↓
订单表      计算pending_qty    按物料分组    关联订单      scheduled_qty++
(sales_     (quantity -        MOQ检查       设置参数      (防止重复排程)
orders)     scheduled_qty)     生成任务      插入数据库
```

### 关键数据流

**1. 订单→排程关联**
```java
ScheduleCoating coating = new ScheduleCoating();
coating.setOrderItemId(order.getOrderItemId());  // 关联订单明细
coating.setOrderId(order.getOrderId());          // 关联订单
coating.setOrderNo(order.getOrderNo());          // 订单号
coating.setMaterialCode(order.getMaterialCode()); // 从订单获取料号
coating.setMaterialName(order.getMaterialName()); // 从订单获取产品名
coating.setPlanSqm(order.getPendingArea());       // 从订单获取面积
```

**2. 已排程数量更新**
```sql
UPDATE sales_order_items 
SET scheduled_qty = COALESCE(scheduled_qty, 0) + #{quantity} 
WHERE id = #{id}
```

**3. MOQ检查逻辑**
```java
if (totalArea < config.getMinProductionArea()) {
    System.out.println("总面积小于MOQ - 警告但继续");
}
```

## ⚠️ 注意事项

### 1. MOQ管理
- 当前实现：**警告但允许排程**
- 如需强制限制，修改 `autoScheduleCoating` 方法：
```java
if (totalArea.compareTo(config.getMinProductionArea()) < 0) {
    throw new RuntimeException("总面积不足MOQ，无法排程");
}
```

### 2. 薄膜库存检查
- 当前实现：**不自动锁定薄膜库存**
- 需手动调用薄膜宽度选择功能锁定物料
- 可集成薄膜锁定逻辑到自动排程

### 3. 排程时间计算
- 需要物料配置表有完整的时间参数
- 公式：planDuration = setupTime + (area × unitTime)

### 4. 重复排程防止
- 通过 scheduled_qty 字段控制
- 只排程 pending_qty > 0 的订单
