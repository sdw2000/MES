## ✨ 完整答案

### 🔢 计算公式（最核心）

```
planSqm(m²) = (涂布长度(m) × 母卷宽度(mm)) / 1,000,000
```

### 📊 数据来源

| 参数 | 来源 | 计算方式 | 原值 | 改进后 |
|------|------|--------|------|--------|
| **涂布长度** | 订单明细 | Σ(订单长度×数量) | ❌ 固定5000m | ✅ 动态计算 |
| **母卷宽度** | 订单明细 | MAX(订单宽度) | ❌ 固定1200mm | ✅ 订单最大宽度 |
| **平方米** | 公式计算 | (长×宽)/1000000 | ✅ 正确 | ✅ 保持不变 |

### 📈 改进内容

**改进1：涂布长度从硬编码改为动态计算**

```
总涂布长度(mm) = Σ(订单长度 × 订单数量)

示例：
  订单1：1000m × 100卷 = 100,000m
  订单2：2000m × 50卷  = 100,000m
  ─────────────────────────────────
  总计：200,000m = 200,000,000mm
```

**改进2：母卷宽度从硬编码改为订单最大值**

```
母卷宽度(mm) = MAX(订单宽度)

示例：
  订单1宽度：1200mm
  订单2宽度：1200mm
  订单3宽度：1000mm
  ────────────────
  最大值：1200mm
```

**改进3：最终面积计算**

```
planSqm = (200,000m × 1200mm) / 1,000,000
        = 240.00 m²
```

## 🔍 详细计算过程

### 完整流程示例

```
【输入：3个待排程订单】

PO-001：长度=1000m, 宽度=1200mm, 数量=100卷
PO-002：长度=2000m, 宽度=1200mm, 数量=50卷
PO-003：长度=500m,  宽度=1000mm, 数量=100卷


【第1步：按料号分组】

假设PO-001和PO-002是同一料号(A001)，PO-003是料号(A002)：

分组1：料号A001 → [PO-001, PO-002]
分组2：料号A002 → [PO-003]


【第2步：为分组1计算涂布长度】

料号A001的订单：
  item1.length = 1,000,000 mm (= 1000m)
  item1.rolls  = 100
  item1小计 = 1,000,000 × 100 = 100,000,000 mm

  item2.length = 2,000,000 mm (= 2000m)
  item2.rolls  = 50
  item2小计 = 2,000,000 × 50 = 100,000,000 mm

总长度 = 100,000,000 + 100,000,000 = 200,000,000 mm
转换为米 = 200,000,000 ÷ 1000 = 200,000 m


【第3步：为分组1确定母卷宽度】

料号A001的订单宽度：
  item1.width = 1200 mm
  item2.width = 1200 mm

最大宽度 = max(1200, 1200) = 1200 mm


【第4步：为分组1计算面积】

planSqm = (200,000 m × 1200 mm) / 1,000,000
        = 200,000 × 1200 / 1,000,000
        = 240,000,000 / 1,000,000
        = 240.00 m²


【第5步：为分组2计算（相同过程）】

料号A002的订单：
  item3.length = 500,000 mm (= 500m)
  item3.rolls  = 100
  小计 = 500,000 × 100 = 50,000,000 mm

总长度 = 50,000,000 mm = 50,000 m
最大宽度 = 1000 mm

planSqm = (50,000 × 1000) / 1,000,000
        = 50.00 m²


【输出结果】

涂布任务1 (料号A001)：
  ├─ 涂布长度：200,000 m ✓
  ├─ 母卷宽度：1200 mm ✓
  └─ 计划面积：240.00 m² ✓

涂布任务2 (料号A002)：
  ├─ 涂布长度：50,000 m ✓
  ├─ 母卷宽度：1000 mm ✓
  └─ 计划面积：50.00 m² ✓
```

## 💻 代码实现

### 核心方法位置

**文件**：`ProductionScheduleServiceImpl.java`

**方法1：计算总涂布长度**
```java
private BigDecimal calculateTotalLength(List<Map<String, Object>> items) {
    BigDecimal totalLength = BigDecimal.ZERO;
    
    for (Map<String, Object> item : items) {
        // 获取订单长度（毫米）
        BigDecimal length = parseLength(item.get("length"));
        
        // 获取订单数量（卷数）
        Integer rolls = parseRolls(item.get("rolls"));
        
        // 累加：length(mm) × rolls
        totalLength = totalLength.add(length.multiply(new BigDecimal(rolls)));
    }
    
    return totalLength;
}
```

**方法2：确定母卷宽度**
```java
private Integer determineJumboWidth(List<Map<String, Object>> items) {
    Integer maxWidth = 0;
    
    for (Map<String, Object> item : items) {
        Integer width = parseWidth(item.get("width"));
        if (width > maxWidth) {
            maxWidth = width;
        }
    }
    
    // 优先使用订单宽度，默认1200mm
    return maxWidth > 0 ? maxWidth : 1200;
}
```

**方法3：计算平方米**
```java
// 在generateCoatingTasks()中
BigDecimal totalLength = calculateTotalLength(items);
int coatingLengthMeters = totalLength.divide(new BigDecimal(1000), 0, BigDecimal.ROUND_HALF_UP).intValue();
Integer jumboWidth = determineJumboWidth(items);

BigDecimal planSqm = new BigDecimal(coatingLengthMeters)
    .multiply(new BigDecimal(jumboWidth))
    .divide(new BigDecimal(1000000), 2, BigDecimal.ROUND_HALF_UP);

coating.setPlanSqm(planSqm);
```

## 🔧 常见问题解答

### Q1：为什么要改成动态计算？

**A**：
- ✅ 固定值不符合实际业务需求
- ✅ 涂布数量应根据订单数据动态计算
- ✅ 不同订单可能有不同宽度
- ✅ 动态计算更灵活，更科学

### Q2：如果没有订单宽度怎么办？

**A**：
系统有三级降级方案：
```
优先级1：使用订单中最大宽度
优先级2：使用设备配置宽度（可选）
优先级3：使用系统默认值 1200mm
```

### Q3：单位是怎么转换的？

**A**：
```
数据库存储：毫米(mm)
计算过程：毫米转米 ÷ 1000
最终结果：平方米(m²)

公式：(长(m) × 宽(mm)) / 1,000,000
     = (长(m) × 宽(mm)) / 1,000,000
     = (长(mm)÷1000 × 宽(mm)) / 1,000,000
```

### Q4：如何验证计算正确？

**A**：
```sql
-- 验证SQL
SELECT 
  task_no,
  plan_length,
  jumbo_width,
  plan_sqm,
  (plan_length * jumbo_width / 1000000) as calculated,
  CASE 
    WHEN ABS(plan_sqm - ROUND(plan_length * jumbo_width / 1000000, 2)) < 0.01 
    THEN '✓ 正确'
    ELSE '✗ 错误'
  END as status
FROM schedule_coating;
```
