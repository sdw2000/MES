## 【核心功能特性】

### 1. 10分钟时间精度处理 ✅
```
算法: 
  rounded_minute = (current_minute + 5) / 10 * 10

示例:
  14:23 → 14:20 (23+5=28, 28/10*10=20)
  14:26 → 14:30 (26+5=31, 31/10*10=30)
  15:55 → 16:00 (55+5=60, 60/10*10=60 → +1小时)

实现位置: 
  ProductionScheduleServiceImpl.roundTimeToTenMinutes()

应用时机:
  ✓ 新增任务时自动应用
  ✓ 编辑任务时自动应用
  ✓ 开始/完成任务时保留原值
```

### 2. 任务号自动生成 ✅
```
格式: SLT-YYYYMMDD-XXXXX

示例:
  2024-12-20 第一个任务: SLT-20241220-00001
  2024-12-20 第二个任务: SLT-20241220-00002
  2024-12-21 第一个任务: SLT-20241221-00001

实现位置:
  ProductionScheduleServiceImpl.generateSlittingTaskNo()

数据库支撑:
  SELECT MAX(task_no) FROM schedule_slitting 
  WHERE task_no LIKE 'SLT-YYYYMMDD-%'
```

### 3. 规格自动生成 ✅
```
格式: 料号-宽度mm x 长度m x 卷数卷

示例:
  FT-001-50mm x 100m x 100卷
  FT-002-38mm x 80m x 50卷

实现位置:
  ProductionScheduleServiceImpl.addSlittingTask()
  String.format("%s-%smm x %sm x %d卷", ...)

收集的信息:
  materialCode (料号)
  targetWidth (目标宽度/mm)
  slitLength (分切长度/m)
  planRolls (计划卷数)
```

### 4. 订单完整追踪 ✅
```
新增字段支撑:
  order_id        - 订单主记录ID
  order_no        - 订单号(支持查询)
  order_item_id   - 订单明细ID
  order_detail_no - 订单详情号

功能:
  ✓ 能追踪任务来源于哪张订单
  ✓ 支持按订单号查询所有相关任务
  ✓ 便于订单交付状态管理
```

### 5. 操作人员责任追踪 ✅
```
新增字段:
  operator_id   - 操作人员ID
  operator_name - 操作人名称

功能:
  ✓ 记录谁执行了该分切任务
  ✓ 便于生产绩效统计
  ✓ 支持责任追溯
```

### 6. 完整的CRUD操作 ✅
```
Create: POST /api/production/schedule/slitting/add
        - 自动生成任务号
        - 自动生成规格
        - 应用时间精度
        - 返回完整的任务信息

Read:   GET /api/production/schedule/slitting/list
        - 分页查询(pageNum, pageSize)
        - 多条件过滤(orderNo, status, planDate, equipmentId)
        - 排序(按创建时间倒序)

Update: PUT /api/production/schedule/slitting/update
        - 编辑任何字段
        - 应用时间精度处理
        - 返回更新结果

Delete: DELETE /api/production/schedule/slitting/delete/{id}
        - 删除指定任务
        - 支持逻辑删除
```

### 7. 状态管理流程 ✅
```
状态流转:
  pending → in_progress → completed
    (待生产)   (进行中)    (已完成)

关键字段:
  status              - 任务状态
  actualStartTime     - 实际开始时间(开始任务时填充)
  actualEndTime       - 实际结束时间(完成任务时填充)

对应API:
  POST /slitting/start/{id}     - status: "in_progress"
  POST /slitting/complete/{id}  - status: "completed"
```

    "operatorName": "张三",

Response:
  {
    "code": 200,
    "message": "success",
    "data": {
      "id": 123,
      "taskNo": "SLT-20241220-00001",        ← 自动生成
      "orderNo": "ORD-2024-001",
      "spec": "FT-001-50mm x 100m x 100卷", ← 自动生成

重要: 后端会自动处理以下逻辑
  ✓ 生成任务号 (SLT-YYYYMMDD-XXXXX)
  ✓ 生成规格 (料号-宽度-长度-卷数)
  ✓ 时间精度 (14:23 → 14:20)
  ✓ 设置初始状态 (pending)
