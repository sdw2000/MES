## 【关键业务逻辑实现】

### 1. 10分钟时间精度
```
输入: 2024-12-20 14:23:00
处理: (23 + 5) / 10 * 10 = 20
输出: 2024-12-20 14:20:00
```

### 2. 任务号自动生成
```
格式: SLT-YYYYMMDD-XXXXX
示例: SLT-20241220-00001
      SLT-20241220-00002
      SLT-20241221-00001
```

### 3. 规格自动生成
```
输入: materialCode=FT-001, targetWidth=50, slitLength=100, planRolls=50
输出: FT-001-50mm x 100m x 50卷
```

## 【技术支持】

### 常见问题

**Q1: 编译失败提示找不到ProcessParams方法?**
A: 这不是我们修改引入的错误,是项目中ProcessParams实体类缺少getter/setter。解决:
- 使用 `@Data` 注解替代 getter/setter
- 或修复ProcessParams类的字段定义

**Q2: 前端路由404?**
A: 检查:
- 路由配置是否正确(已在router/index.js中)
- SlittingTasksNew.vue是否存在
- slittingTasks.js是否引入正确

**Q3: 时间保存到数据库后变成NULL?**
A: 检查:
- ScheduleSlitting实体中plan_start_time是否有@JsonFormat注解
- MySQL字段类型是否为DATETIME
- 数据库迁移脚本是否执行

**Q4: 能否只部署前端不修改后端?**
A: 不行。前端API调用需要后端提供对应的REST接口。必须:
1. 先部署后端(添加6个新API端点)
2. 再部署前端(使用新组件)
