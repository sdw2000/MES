# 涂布排程逻辑优化方案

## 📋 当前问题

### 1. **数据缺失问题**
- ✗ 料号信息为空
- ✗ 订单面积为空  
- ✗ 产品信息不完整

### 2. **排程逻辑问题**
- ✗ 涂布计划没有关联销售订单数据

## 🎯 优化目标

### 1. **数据完整性**
从销售订单中获取完整的产品信息：
- 料号（materialCode）
- 产品名称（materialName）
- 订单号（orderNo）
- 订单数量/面积
- 客户信息
- 交期

### 2. **排程逻辑优化**
```
销售订单 → 按料号分组 → 计算总需求 → 考虑MOQ → 生成涂布计划
```

### 3. **待排程订单管理**
- 显示所有未排程的订单明细
- 按料号、交期、客户等级排序
- 支持批量选择排程
- 自动计算合并排程

## 🔧 技术实现方案

### 方案A：修复现有排程逻辑（推荐）

#### 1. 修改排程创建流程

**当前流程：**
```
手动创建涂布计划 → 填写基本信息 → 保存
```

**优化流程：**
```
查看待排程订单 → 选择订单 → 按料号自动分组 → 生成涂布计划
```

#### 2. 数据库表关联

确保 `schedule_coating` 表关联到订单：

```sql
-- schedule_coating 应该包含以下字段

ALTER TABLE schedule_coating ADD COLUMN IF NOT EXISTS order_id BIGINT COMMENT '订单ID';

-- 创建外键
ALTER TABLE schedule_coating 
ADD CONSTRAINT fk_coating_order_item
```

#### 4. 按料号分组排程

**API：** `POST /api/production/schedule/auto-schedule-by-material`

**请求参数：**
```json
{
  "materialCode": "1011-R02-1507-B01-0400",
  "orderItemIds": [1001, 1002, 1003],
  "scheduleDate": "2026-01-16",
  "equipmentCode": "TB-001",
  "mergeOrders": true
}
```

**逻辑：**
1. 查询该料号的所有待排程订单
2. 计算总需求面积
3. 检查物料最小生产量
4. 如果总需求 < MOQ，调整为MOQ
5. 创建涂布计划
6. 关联所有订单明细
7. 更新订单的已排程数量

#### 5. 物料最小生产量(MOQ)管理

**新增配置表：**
```sql
CREATE TABLE IF NOT EXISTS material_production_config (
  id BIGINT PRIMARY KEY AUTO_INCREMENT,
  material_code VARCHAR(50) NOT NULL COMMENT '物料编号',
  material_name VARCHAR(100) COMMENT '物料名称',
  min_production_qty INT COMMENT '最小生产数量',
  min_production_area DECIMAL(10,2) COMMENT '最小生产面积(㎡)',
  standard_batch_size INT COMMENT '标准批量',
  setup_time INT COMMENT '调机时间(分钟)',
  unit_time DECIMAL(10,2) COMMENT '单位时间(分钟/㎡)',
  remark VARCHAR(500),
  create_time DATETIME DEFAULT CURRENT_TIMESTAMP,
  update_time DATETIME DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP,
  UNIQUE KEY uk_material_code (material_code)
) ENGINE=InnoDB COMMENT='物料生产配置表';
```

### 3. 查看现有订单数据

```sql
-- 查看待排程的订单
SELECT 
    so.order_no,
    soi.id as order_item_id,
    soi.material_code,
    soi.material_name,
    soi.quantity,
    soi.scheduled_quantity,
    (soi.quantity - IFNULL(soi.scheduled_quantity, 0)) as pending_qty,
    so.customer_name,
    so.delivery_date
FROM sales_order_item soi
INNER JOIN sales_order so ON soi.order_id = so.id
WHERE soi.quantity > IFNULL(soi.scheduled_quantity, 0)
  AND so.status != 'cancelled'
ORDER BY soi.material_code, so.delivery_date;
```
