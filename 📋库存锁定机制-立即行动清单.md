## ⚡ 快速问题排查

### 问题 1: MySQL 脚本执行失败
**症状**: "Access denied" 或 "Unknown database"

**解决**:
```bash
# 确认 MySQL 账号
mysql -u root -p  # 输入正确的密码

# 确认数据库存在
SHOW DATABASES;
# 应该看到 MES_database

# 如果不存在，创建
CREATE DATABASE MES_database;
```

### 问题 2: 编译失败 - 找不到类
**症状**: "error: cannot find symbol"

**解决**:
```bash
# 重新下载所有依赖
mvn clean install -DskipTests

# 等待下载完成后再次编译
mvn compile
```

### 问题 3: 后端启动失败 - 端口被占用
**症状**: "Address already in use: bind"

**解决**:
```bash
# 查看占用 8080 端口的进程
netstat -ano | findstr :8080

# 停止该进程
taskkill /PID <PID> /F

# 或使用不同的端口
java -jar target/MES.jar --server.port=8081
```

### 问题 4: 前端页面 404
**症状**: "Cannot find module" 或 "404 Not Found"

**解决**:
1. 检查路由配置是否正确
2. 检查 materialLock.vue 文件是否存在
3. 重启前端服务
4. 清除浏览器缓存 (Ctrl + Shift + Delete)
