# 涂布排程锁定和复卷排程实现方案

## 需求概述

当订单被计入涂布排程后，需要实现以下功能：
1. **数量锁定**：已排程到涂布的数量被锁定，不能重复排程
2. **日期回填**：已排程订单的涂布日期显示为涂布排程表中的日期
3. **待排程数量更新**：订单排程表中显示剩余待排程数量（原数量 - 已排程数量）
4. **复卷排程**：按涂布日期排序，对已完成涂布的订单进行复卷排程

## 实现方案

### 1. 数据库设计

#### manual_schedule 表新增字段
```sql
ALTER TABLE manual_schedule 
ADD COLUMN rewinding_scheduled_area DECIMAL(10, 2) DEFAULT 0 COMMENT '复卷已排程面积(㎡)';
```

#### 关键字段说明
- `coating_area`: 涂布总面积（㎡）
- `rewinding_scheduled_area`: 已排入复卷的面积（㎡）
- 剩余待复卷面积 = `coating_area - rewinding_scheduled_area`
- `status`: 
  - `PENDING`: 待处理
  - `COATING_SCHEDULED`: 已排涂布
  - `REWINDING_SCHEDULED`: 已排复卷（当剩余面积为0时）

### 2. 后端实现

#### 2.1 Mapper 层 (ManualScheduleMapper.java)

**查询待排程订单**（已实现）

```java
        "(soi.rolls - IFNULL(soi.scheduled_qty, 0)) AS remaining_qty, " +
        "ms.coating_date, " +  // 从 manual_schedule 回填涂布日期
        "...")
List<Map<String, Object>> selectPendingOrders();
```

**查询已完成涂布待复卷的订单**（✅ 新增）
```java
@Select("SELECT " +
        "ms.id AS schedule_id, " +
        "ms.coating_date, " +
        "ms.coating_area, " +
        "IFNULL(ms.rewinding_scheduled_area, 0) AS rewinding_scheduled_area, " +
        "(ms.coating_area - IFNULL(ms.rewinding_scheduled_area, 0)) AS remaining_coating_area, " +
        "... " +
        "FROM manual_schedule ms " +
        "WHERE ms.status = 'COATING_SCHEDULED' " +
        "AND (ms.coating_area - IFNULL(ms.rewinding_scheduled_area, 0)) > 0 " +
        "ORDER BY ms.coating_date ASC, o.delivery_date ASC")
List<Map<String, Object>> selectCoatingCompletedOrders();
```

**更新复卷已排程面积**（✅ 新增）
```java
@Update("UPDATE manual_schedule " +
        "SET rewinding_scheduled_area = IFNULL(rewinding_scheduled_area, 0) + #{area}, " +
        "status = CASE WHEN (coating_area - IFNULL(rewinding_scheduled_area, 0) - #{area}) <= 0 " +
        "     THEN 'REWINDING_SCHEDULED' ELSE status END " +
        "WHERE id = #{scheduleId}")
int updateRewindingScheduledArea(@Param("scheduleId") Long scheduleId, @Param("area") BigDecimal area);
```

#### 3.2 新增数据和方法

```javascript
data() {
  return {
    rewindingLoading: false,
    rewindingList: [],
    // ...
  }
},

methods: {
  // 加载待复卷订单列表
  async loadRewindingOrders() {
    this.rewindingLoading = true
    try {
      const res = await getCoatingCompletedOrders()
      if (res.code === 200 || res.code === 20000) {
        this.rewindingList = res.data || []
      }
    } catch (error) {
      this.$message.error('加载待复卷订单失败')
    } finally {
      this.rewindingLoading = false
    }
  },
  
  // 确认复卷排程
  async handleConfirmRewinding(row) {
    if (!row.rewinding_date) {
      this.$message.warning('请选择复卷日期')
      return
    }
    
    this.$confirm(`确认将涂布面积 ${row.remaining_coating_area}㎡ 排入复卷？`, '提示', {
      confirmButtonText: '确定',
      cancelButtonText: '取消',
      type: 'info'
    }).then(async() => {
      try {
        const res = await createRewindingSchedule({
          scheduleId: row.schedule_id,
          rewindingArea: row.remaining_coating_area,
          rewindingDate: row.rewinding_date
        })
        
        if (res.code === 200 || res.code === 20000) {
          this.$message.success('复卷排程确认成功')
          this.loadRewindingOrders()
        }
      } catch (err) {
        this.$message.error('排程确认失败: ' + err.message)
      }
    })
  },
  
  // Tab 切换时加载数据
  onTabChange(tab) {
    if (tab.name === 'coating') {
      this.loadCoatingSchedules()
    } else if (tab.name === 'rewinding') {
      this.loadRewindingOrders()
    }
  }
}
```

### 4. 业务流程

#### 流程 1: 订单排程 → 涂布排程
```
1. 用户在"订单排程"Tab选择订单
2. 点击"计算涂布"，系统计算需要的涂布面积
3. 填写涂布日期、选择机台
4. 点击"确认涂布"：
   - 创建涂布排程记录（status='COATING_SCHEDULED'）
   - 更新 sales_order_items.scheduled_qty（锁定数量）
   - 回填 sales_order_items.coating_date
5. 订单排程表中该订单的 remaining_qty 减少
```

#### 流程 2: 涂布排程 → 复卷排程
```
1. 切换到"复卷排程"Tab
2. 系统显示已完成涂布的订单（status='COATING_SCHEDULED'）
3. 订单按涂布日期升序排列（coating_date ASC）
4. 显示每个订单的：
   - 涂布日期（醒目标识）
   - 涂布面积（coating_area）
   - 已排复卷面积（rewinding_scheduled_area）
   - 待复卷面积（remaining_coating_area）
5. 用户选择复卷日期，点击"确认复卷"：
   - 更新 manual_schedule.rewinding_scheduled_area
   - 当 remaining_coating_area = 0 时，status 变为 'REWINDING_SCHEDULED'
```

## 数据一致性保证

### 订单数量锁定
```
原订单数量（order_qty）
- 已完成数量（completed_qty）
- 已排程数量（scheduled_qty）
= 待排程数量（remaining_qty）
```

### 涂布到复卷的面积追踪
```
涂布总面积（coating_area）
- 已排复卷面积（rewinding_scheduled_area）
= 待复卷面积（remaining_coating_area）
```

### 状态流转
```
订单创建 → PENDING
  ↓
确认涂布 → COATING_SCHEDULED（同时锁定 scheduled_qty）
  ↓
确认复卷 → REWINDING_SCHEDULED（当 remaining_coating_area = 0）
  ↓
完成生产 → COMPLETED
```

## 注意事项

1. **并发控制**：涂布面积锁定时应考虑并发场景，建议使用乐观锁或数据库事务
2. **数据校验**：确认复卷时需校验 remaining_coating_area > 0
3. **日期顺序**：复卷日期应 >= 涂布日期
4. **状态一致性**：状态流转要保证单向且不可逆（除非回退操作）
5. **分页支持**：如果数据量大，需要添加分页功能
