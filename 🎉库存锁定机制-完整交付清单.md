## 🔑 核心功能一览

### 功能 1: 物料锁定
**API**: `POST /production/schedule-material/lock/{scheduleId}`

**流程**:
1. 查询排程的所有订单
2. 对每个订单执行三层级物料查询：
   - Level 1: 查复卷（优先级最高）
   - Level 2: 查母卷（优先级中等）
   - Level 3: 触发涂布（优先级最低）
3. 创建锁定和分配记录
4. 返回锁定统计结果

**返回示例**:
```json
{
  "success": true,
  "totalOrders": 5,
  "fullyLockedOrders": 3,
  "partiallyLockedOrders": 1,
  "unlockedOrders": 1,
  "totalRequiredArea": "1000.00",
  "totalAllocatedArea": "850.00",
  "needCoatingOrderIds": [456]
}
```

### 功能 2: 生产领料
**API**: `POST /production/schedule-material/allocate`

**流程**:
1. 接收锁定记录 IDs
2. 更新锁定状态为 "已领料"
3. 扣减库存（updated consumed_area）
4. 记录领料时间和操作人

### 功能 3: 释放锁定
**API**: `POST /production/schedule-material/release/{scheduleId}`

**流程**:
1. 查询排程的所有锁定记录
2. 更新状态为 "已释放"
3. 恢复库存（reset reserved_area）
4. 记录释放时间

### 功能 4: 查询接口 (4 个)

#### 查询 1: 排程分配情况
```
GET /production/schedule-material/allocation/{scheduleId}
返回: List<ScheduleMaterialAllocation>
```

#### 查询 2: 订单分配情况
```
GET /production/schedule-material/allocation/{scheduleId}/{orderId}
返回: ScheduleMaterialAllocation
```

#### 查询 3: 物料被锁定情况
```
GET /production/schedule-material/tape-locks/{tapeId}
返回: List<ScheduleMaterialLock>
```

#### 查询 4: 订单锁定记录
```
GET /production/schedule-material/order-locks/{orderId}
返回: List<ScheduleMaterialLock>
```

## 📊 技术特点

### 1. 高并发处理
- ✅ **乐观锁**: 使用 version 字段防止并发冲突
- ✅ **事务管理**: @Transactional 保证一致性
- ✅ **原子操作**: 库存更新在同一事务内

### 2. 灵活的物料层级
- ✅ **三层级查询**: 复卷 → 母卷 → 涂布
- ✅ **自适应分配**: 根据库存自动选择物料来源
- ✅ **涂布触发**: 自动识别需要涂布的订单

### 3. 完整的状态管理
- ✅ **五种状态**: 锁定中 → 已领料 → 已消耗 / 已释放
- ✅ **时间戳**: 完整记录每个状态转移的时间
- ✅ **操作人**: 记录执行各操作的用户

### 4. 精确的面积计算
- ✅ **按面积锁定**: 而不是按卷数
- ✅ **精确扣减**: 按实际领料面积扣库存
- ✅ **面积追踪**: 可用/锁定/消耗三个维度

### 5. 清晰的 API 设计
- ✅ **RESTful 风格**: 标准的 HTTP 方法和路由
- ✅ **统一响应**: 所有 API 返回一致的响应格式
- ✅ **完整文档**: 每个 API 都有详细的说明

## ⚠️ 已知限制

1. **并发控制**
   - 使用乐观锁，高并发下可能有重试
   - 建议最多 5 个排程同时启动

2. **涂布排程**
   - 目前只标记需要涂布，不自动创建涂布排程
   - 需要手动在 UI 中创建涂布排程

3. **数据清理**
   - 没有自动清理旧的锁定记录
   - 建议定期手动或通过脚本清理

4. **权限控制**
   - 所有 API 都需要有效的登录
   - 没有精细的权限控制

## 🚨 常见问题

### Q1: 如何处理锁定冲突？
**A**: 框架已处理乐观锁冲突，会自动重试查询和锁定。如果多次重试失败，可能是库存确实不足。

### Q2: 如何修改锁定规则？
**A**: 修改 ScheduleMaterialLockingServiceImpl 中的查询条件或优先级逻辑。

### Q3: 如何添加新的物料类型？
**A**: 在数据库中添加新的 reel_type 值，并在 lockFromReels 方法中添加相应的查询。

### Q4: 数据库连接失败怎么办？
**A**: 检查 application.properties 中的数据库连接配置，确保用户名、密码、数据库名正确。

### Q5: 前端页面显示空白？
**A**: 检查浏览器控制台 (F12) 中的错误信息，通常是路由配置或 API 路径错误。
