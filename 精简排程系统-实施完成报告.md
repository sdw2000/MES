## 🔑 核心功能说明

### 1. 客户优先级评分算法

```java
总分 = 新客户分 + 账期分 + 月均交易额分 + 单价偏差分

其中：
• 新客户分 = 20（is_new_customer=1） 或 0
• 账期分 = +10(预付) / 0(月结30) / -10(月结60) / -20(月结90)
• 月均额分 = 近3个月月均交易额 / 30,000（每3万贡献1分）
• 单价偏差分 = (客户历史均价 / 行业均价 - 1) × 50
```

**示例：**
- 新客户 + 预付 + 月均10万 + 单价高10% → 20 + 10 + 3.33 + 5 = **38.33分**
- 老客户 + 月结60 + 月均5万 + 单价低5% → 0 - 10 + 1.67 - 2.5 = **-10.83分**

### 2. 库存三级匹配流程

```
订单需求 5000㎡ → 
    ↓
【第一级】查找成品库存（分切卷）
    料号完全匹配 + rollType='分切卷' + availableArea > 0
    找到 2000㎡ 成品 → 锁定 → 剩余需求 3000㎡
    ↓
【第二级】查找复卷库存（复卷）
    料号前缀匹配 + rollType='复卷' + availableArea > 0
    找到 1500㎡ 复卷 → 锁定 → 剩余需求 1500㎡
    ↓
【第三级】查找母卷库存（母卷）
    料号前缀匹配 + rollType='母卷' + availableArea > 0
    找到 1000㎡ 母卷 → 锁定 → 剩余需求 500㎡
    ↓
【缺口处理】需要涂布生产 500㎡
```

### 3. 涂布排程规则

**颜色分组：**
```
订单A: FT-001-38mm (红色) 3000㎡
订单B: FT-002-38mm (红色) 2000㎡
订单C: FT-003-50mm (蓝色) 4000㎡

→ 合并为2个批次：
  批次1: 红色 5000㎡ (A+B)
  批次2: 蓝色 4000㎡ (C)
```

**MOQ检查：**
```
如果批次总量 < MOQ:
  → needConfirmation = true
  → 提示："未达MOQ，是否继续？" 或 "推荐合并其他订单"
  
如果 manualConfirmed = true:
  → 继续排程
否则:
  → 加入待确认列表
```

**设备分配：**
```
3台涂布机并行：
  coating-1: 最近任务结束于 2024-12-20 10:00
  coating-2: 最近任务结束于 2024-12-20 08:30  ← 最早可用
  coating-3: 最近任务结束于 2024-12-20 11:15

批次1分配到 coating-2，开始时间 = 2024-12-20 08:30 + 换单时间
```

### 4. 设备并行调度

```
涂布（3台）    复卷（5台）      分切（10台）
   ↓              ↓                ↓
coating-1  →  rewinding-1  →  slitting-1
coating-2  →  rewinding-2  →  slitting-2
coating-3  →  rewinding-3  →  slitting-3
               rewinding-4  →  slitting-4
               rewinding-5  →  slitting-5
                               slitting-6
                               ...
                               slitting-10

依赖关系：
  涂布完成 → 复卷开始
  复卷完成 → 分切开始
  分切完成 → 预计交付时间
```

### 5. 交付时间预测

```
订单 SO-001:
  涂布: 2024-12-20 08:30 → 2024-12-20 12:00 (3.5小时)
  复卷: 2024-12-20 12:20 → 2024-12-20 14:30 (2.2小时)
  分切: 2024-12-20 14:50 → 2024-12-20 16:00 (1.2小时)
  预计交付: 2024-12-20 16:00

客户要求: 2024-12-25
延期天数: -5天 (提前)
风险等级: LOW
```

```java
@Configuration
@EnableScheduling
public class ScheduleConfig {
    
    @Autowired
    private CustomerTransactionSummaryMapper transactionMapper;
    
    @Autowired
    private CustomerMaterialStatsMapper materialStatsMapper;
    
    @Autowired
    private MaterialBasePriceMapper basePriceMapper;
    
    /**
     * 每天凌晨1点刷新统计数据
     */
    @Scheduled(cron = "0 0 1 * * ?")
    public void refreshStatistics() {
        log.info("开始刷新统计数据...");
        
        transactionMapper.refreshAllStats();
        materialStatsMapper.refreshAllStats();
        basePriceMapper.refreshAllStats();
        
        log.info("✓ 统计数据刷新完成");
    }
}
```
