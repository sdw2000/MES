# 库存锁定机制 - 集成测试用例

本文档定义了库存锁定机制的完整集成测试用例。

**测试目标**: 验证物料锁定的完整流程是否正常工作  
**预计时间**: 2-3 小时  
**执行者**: QA 测试员或开发人员

## 🔧 前置条件

### 环境准备
- [ ] 后端服务已启动，运行在 `http://localhost:8080`
- [ ] 前端服务已启动，运行在 `http://localhost:8000`
- [ ] MySQL 数据库已连接，包含测试数据
- [ ] 所有数据库表已创建（schedule_material_lock, schedule_material_allocation）

### 数据准备

执行以下 SQL 脚本准备测试数据：

```sql
-- 1. 清空测试数据
DELETE FROM schedule_material_lock WHERE schedule_id = 999;
DELETE FROM schedule_material_allocation WHERE schedule_id = 999;
DELETE FROM schedule_material_lock WHERE schedule_id = 1000;
DELETE FROM schedule_material_allocation WHERE schedule_id = 1000;

-- 2. 插入测试排程 (如果不存在)
INSERT INTO batch_schedule (id, schedule_code, status, created_at)
VALUES (999, 'TEST_SCHEDULE_001', 'pending', NOW())
ON DUPLICATE KEY UPDATE updated_at = NOW();

-- 3. 插入测试订单
INSERT INTO sales_order (id, order_no, schedule_id, thickness, width, required_area, created_at)
VALUES 
  (9991, 'TEST_ORDER_001', 999, 10, 500, 100.00, NOW()),
  (9992, 'TEST_ORDER_002', 999, 10, 500, 150.00, NOW()),
  (9993, 'TEST_ORDER_003', 999, 10, 500, 200.00, NOW())
ON DUPLICATE KEY UPDATE updated_at = NOW();

-- 4. 确保库存有可用物料
UPDATE tape_stock 
SET available_area = COALESCE(total_area, 0) - COALESCE(consumed_area, 0),
    reserved_area = 0,
    version = 1
WHERE status = 'in_stock' AND available_area = 0
LIMIT 10;
```

## 🧪 测试用例

### 用例 1: 基础物料锁定 - 复卷充足

**目标**: 验证当复卷库存充足时的物料锁定流程

**场景**: 排程需要 3 个订单的物料，每个订单 100m²，共 300m²  
**预期**: 所有订单都完全锁定

#### 步骤

1️⃣ **准备测试数据**
```sql
-- 检查是否有符合条件的复卷库存
SELECT id, material_code, thickness, width, available_area, reel_type
FROM tape_stock
WHERE reel_type = '复卷' 
  AND thickness = 10 
  AND width = 500
  AND available_area >= 50
LIMIT 5;
```

如果没有数据，插入测试物料：
```sql
INSERT INTO tape_stock (
  material_code, thickness, width, total_area, available_area,
  reel_type, status, created_at
) VALUES (
  'TEST_REEL_001', 10, 500, 500.00, 500.00,
  '复卷', 'in_stock', NOW()
);
```

2️⃣ **调用物料锁定 API**
```
POST http://localhost:8080/production/schedule-material/lock/999
Content-Type: application/json
```

**预期响应**:
```json
{
  "code": 0,
  "success": true,
  "data": {
    "success": true,
    "totalOrders": 3,
    "fullyLockedOrders": 3,
    "partiallyLockedOrders": 0,
    "unlockedOrders": 0,
    "totalRequiredArea": "450.00",
    "totalAllocatedArea": "450.00",
    "needCoatingOrderIds": []
  }
}
```

3️⃣ **验证数据库状态**
```sql
-- 验证锁定记录是否创建
SELECT COUNT(*) AS lock_count
FROM schedule_material_lock
WHERE schedule_id = 999;
-- 预期: 3 条记录

-- 验证分配记录是否创建
SELECT COUNT(*) AS allocation_count, allocation_status
FROM schedule_material_allocation
WHERE schedule_id = 999
GROUP BY allocation_status;
-- 预期: 3 条 "完全满足" 的记录

-- 验证库存是否更新
SELECT id, available_area, reserved_area
FROM tape_stock
WHERE id IN (
  SELECT tape_stock_id FROM schedule_material_lock
  WHERE schedule_id = 999
);
-- 预期: available_area 减少
```

#### 验收标准
- ✅ API 返回 success = true
- ✅ fullyLockedOrders = 3
- ✅ totalAllocatedArea ≈ 450m²
- ✅ 数据库中有 3 条 schedule_material_lock 记录
- ✅ 库存的 reserved_area 已更新

### 用例 2: 复卷不足，使用母卷

**目标**: 验证当复卷不足时自动使用母卷的流程

**场景**: 
- 排程需要 3 个订单，总需求 500m²
- 复卷库存: 200m² (不足)
- 母卷库存: 400m² (充足补充)
- 预期: 复卷优先锁定，母卷补充不足部分

#### 步骤

1️⃣ **准备测试数据**
```sql
-- 清空复卷库存
UPDATE tape_stock
SET available_area = 0
WHERE reel_type = '复卷' AND thickness = 11 AND width = 600;

-- 插入有限的复卷
INSERT INTO tape_stock (
  material_code, thickness, width, total_area, available_area,
  reel_type, status
) VALUES (
  'TEST_REEL_LIMIT', 11, 600, 200.00, 200.00,
  '复卷', 'in_stock'
);

-- 插入充足的母卷
INSERT INTO tape_stock (
  material_code, thickness, width, total_area, available_area,
  reel_type, status
) VALUES (
  'TEST_MOTHER_REEL', 11, 600, 400.00, 400.00,
  '母卷', 'in_stock'
);

-- 创建新的测试排程和订单
INSERT INTO batch_schedule (id, schedule_code) VALUES (1000, 'TEST_SCHEDULE_002');
INSERT INTO sales_order (id, order_no, schedule_id, thickness, width, required_area)
VALUES 
  (10001, 'TEST_ORDER_LVL2_1', 1000, 11, 600, 200.00),
  (10002, 'TEST_ORDER_LVL2_2', 1000, 11, 600, 150.00),
  (10003, 'TEST_ORDER_LVL2_3', 1000, 11, 600, 150.00);
```

2️⃣ **调用物料锁定 API**
```
POST http://localhost:8080/production/schedule-material/lock/1000
```

3️⃣ **验证响应**

**预期响应**:
```json
{
  "code": 0,
  "success": true,
  "data": {
    "success": true,
    "totalOrders": 3,
    "fullyLockedOrders": 3,
    "totalRequiredArea": "500.00",
    "totalAllocatedArea": "500.00",
    "needCoatingOrderIds": []
  }
}
```

4️⃣ **验证锁定记录**
```sql
-- 验证既有复卷锁定，又有母卷锁定
SELECT tape_stock_id, reel_type, SUM(locked_area) as total_locked
FROM schedule_material_lock sml
JOIN tape_stock ts ON sml.tape_stock_id = ts.id
WHERE sml.schedule_id = 1000
GROUP BY reel_type;

-- 预期结果:
-- 复卷: 200m²
-- 母卷: 300m²
```

#### 验收标准
- ✅ API 返回 success = true
- ✅ fullyLockedOrders = 3
- ✅ 同时有复卷和母卷的锁定记录
- ✅ 库存已更新，reserved_area > 0
